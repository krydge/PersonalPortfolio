I wrote a simple Api that wraps another Api. (The Chuck Norris Joke Api)

In the World of programming communication between software is always something to consider. The way we accomplish this is by using API. Application Programming Interface. This is the way that a program or part of a program has agreed to receive requests for Data and the way they will return the requested Data.

We have seen many different types of API’s designs. The ones I have most often seen are REST, and RPC. Each has their own standards and definitions as to what makes them work. I use REST for most of my work. This is nice because it allows me to rely on HTTP. Most of the time it uses GET and POST Requests.

I have made a simple API that wraps another API “ The Chuck Norris Joke API” to return a random joke

I used the Template from Visual Studio to scaffold out an ASP.net API. This gives us a working API to gather weather data.

I then changed the existing code to create a http client.

I set the base url to call the existing API format.

I then added any special requirements to the url.

I make the call to the other API.

This Api returns the data as json.

I have to parse this data and store it in an object.

I then format the Data I received and return it back with just the joke to the user.
